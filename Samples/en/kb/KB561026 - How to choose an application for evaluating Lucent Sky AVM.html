Introduction

This article describes the characteristics of an ideal application for evaluating Lucent Sky AVM.

More Information

This article contains a checklist that is to help you isolate the best application(s) for evaluating Lucent Sky AVM. Successful use of Lucent Sky AVM requires the application be compatible with our software in addition to being buildable. This list will help you choose an application that has the highest likelihood of an expedient and successful evaluation. Troubleshooting for applications with compatibility issues is available, but can cause delays in evaluations. It is suggested that these applications be tested separately or at a later time to the initial evaluation, which should focus on the substance of Application Vulnerability Mitigation (AVM).

All applications

  • Web applications will generally have the best results, as they often contain both the most vulnerabilities (by volume) and the most vulnerabilities Lucent Sky AVM supports. For a full list of vulnerabilities Lucent Sky AVM can automatically fix, visit www.lucentsky.com/avm.

.NET

  • Compatible attributes
    • Most .NET applications are compatible with Lucent Sky AVM.
  • Potentially incompatible attributes
    • .NET 1.0 and 1.1 applications are likely to be incompatible with Lucent Sky AVM without conversion.
  • Notes
    • Some applications use libraries installed by NuGet or installed locally (such as most SDKs) and do not have the referenced libraries in the source code repository. Copy these libraries to the relevant bin directories before uploading the source code to Lucent Sky AVM.

Java

  • Compatible attributes
    • Applications for which all .jsp, .java, and .tag files compile.
    • Applications containing all .jar files necessary for compilation in WEB-INF\lib. This includes third-party applications that are needed during compilation (such as hibernate.jar, servlet-api.jar).
    • Applications that are Java 1.6 compliant.
    • Applications that are compatible with Apache Tomcat 7.0 or newer.
    • Applications that contain exactly one WEB-INF directory, with a deployment descriptor (web.xml) file located directly underneath that directory.
    • Applications that are a single project (i.e. one module).
    • Applications that package all external classes in JAR files (as opposed to placing external classes in folders).
    • Applications for which all source files to be scanned are located in a single root directory that is one of the following:
      • For applications in project format, either a java or src directory immediately under the root directory of the .zip file containing the Java project
      • For applications in WAR format, the WEB-INF\classes directory (so chosen because it is the default export location for Java source files in Eclipse)
      • Another directory that is somewhere under the root directory of the .zip file containing the Java project, and is specified by the user and passed in as a scan argument to Lucent Sky AVM.
  • Potentially incompatible attributes
    • Applications that compiles using source files not located under the specified source root directory.
    • Applications that includes include precompilation steps that move or copy source files from external locations during the compilation process.

PHP

  • Compatible attributes
    • Most PHP applications are compatible with Lucent Sky AVM
  • Potentially incompatible attributes
    • Applications using PHP 5.3 or older are likely to have compatibility issues
    • Applications using some MVC frameworks (such as CodeIgnitor or Phalcon) that are not fully supported by Lucent Sky AVM yet may have sections that cannot be analyzed and mitigated. Analysis and Instant Fixes will be available for the sections that are supported.
