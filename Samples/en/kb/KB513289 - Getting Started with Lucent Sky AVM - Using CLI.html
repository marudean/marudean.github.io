Introduction

Lucent Sky AVM is a system that helps developers and other stakeholders identify and automatically mitigate vulnerabilities in an application's source code. Lucent Sky AVM is composed of three components, CLEAR Engine (the core analysis and mitigation engine), Web UI, and Developer UIs. The Developer UIs include CLI, Visual Studio Extension and Eclipse plug-in.

More Information

Prerequisites for Developers

Developers should be familiar with the following tools:

  • C#, Java or VB.NET languages
    • This guide uses C# (which has a similar syntax with Java and VB.NET) in its examples and screenshots.
  • Bash, Command Prompt or Windows PowerShell

Using CLI

In this section, you will see:

  • A demonstration of how to install and configure Lucent Sky AVM CLI
  • A sample procedure for analyzing and mitigating an application with the Lucent Sky AVM CLI

At the end of this section, you will be able to:

  • Install and configure CLI on a development machine
  • Use CLI to scan an application for vulnerabilities and automatically fixing them
  • Download the secured source code of an application from the Lucent Sky AVM CLI
  • Download the security report of an application from the Lucent Sky AVM CLI

To install and configure AVM CLI

  1. Double click on the Lucent Sky AVM CLI setup file (CLEAR_CLI.exe). The file will automatically extract and create a CLEAR_CLI folder at C:\Program Files\Lucent Sky\CLEAR CLI (or the location you chose during the extraction process).
  2. Navigate to C:\Program Files\Lucent Sky\CLEAR CLI (or the location you chose during the extraction process). Open SkyAnalyzer.Interface.Console.exe with a text editor.
  3. Replace all occurrences of clear.contoso.com with the FQDN, IP address or computer name of an AVM Server, and then save and close the file.

The default configuration makes AVM CLI communicate with an AVM Server using insecure channels. To learn more about how to secure the communication between AVM Server and AVM CLI, click the following article number to view the article in the Lucent Sky Knowledge Base:
    KB448674 Administration of Lucent Sky AVM CLI

To analyze and mitigate the vulnerabilities in an application using Lucent Sky AVM CLI

Lucent Sky AVM CLI has three sets of functions: tools, application and scan. They're the content of Lucent Sky AVM CLI's argument interface. Below is the detail about these sets of functions.

  • Tools
    Enter the following in Command Prompt:
    SkyAnalyzer.Interface.Console.exe --interface tools [--method][--value]
    • Methods:
      Method Description
      base64encode Obtain the base64 encoded value of the user credentials. Use with argument value.
    • Arguments:
      Option Description
      --value Required argument for method base64encode.
      The format is "Email Password AccountType". Use Sql as AccountType for built-in accounts; ActiveDirectory for organizational accounts.
    • Example:
      The command below will generate the base64 encoded string of the built-in account alice@contoso.com with the password password:
      SkyAnalyzer.Interface.Console.exe --interface tools --method base64encode --value "alice@contoso.com password Sql"
  • Application
    Enter the following in Command Prompt:
    SkyAnalyzer.Interface.Console.exe [--credential] --interface tools [--method][--applicationid][option [--applicationname][--applicationframework][-- applicationmitigaterules][--applicationidentificationrules][--applicationvectors]]
    • Methods:
      Method Description
      create Create a new application.
      delete Delete an existing application.
    • Arguments:
      Option Description
      --credential Base64 encoded credentials. You can learn more about generating base64 encoded credentials in the Tools section.
      --applicationid The GUID of the application on AVM. AVM will use this GUID to identify the application.
      --applicationname Required argument for method create.
      The name of the application to be created.
      --applicationframework Required argument for method create.
      The framework of the application. Frameworks supported are DotNet, Java and PHP.
      --applicationidentificationrules Optional argument for method create.
      The types of vulnerabilities to identify, separated by commas. For example, CWE79,CWE89,CWE90. If this argument is not present, identification will be enabled on all types.
      This argument must be used with the argument applicationmitigaterules. Also, rules in argument applicationmitigaterules must also exist in this argument.
      --applicationmitigaterules Optional argument for method create.
      The types of vulnerabilities to mitigate, separated by commas. For example, CWE89,CWE90. If this argument is not present, mitigation will be enabled on all types.
      This argument must be used with the argument applicationidentificationrules. Also, rules in this argument must also exist in argument applicationidentificationrules.
      --applicationvectors Optional argument for method create.
      The types of vectors considered potentially dangerous, separated by commas. For example, Database,WebRequest. If this argument is not present, all vectors will be considered potentially dangerous.
    • Example:
      The command below will create a DotNet (.NET) application named ContosoWeb, with the application ID AAAAAAAA-AAAA-AAAA-AAAA-AAAAAAAAAAAA. Rules CWE79, CWE89 and CWE90 are enabled for identification, with CWE89 and CWE90 enabled for mitigation. Database and WebRequest are the only vectors considered potentially dangerous. k2FsaWNlQGNvbnRvc28uY29t is the base64 encoded string of the built-in account alice@contoso.com with the password password:
      SkyAnalyzer.Interface.Console.exe --credential k2FsaWNlQGNvbnRvc28uY29t --interface application --method create --applicationname "ContosoWeb" --applicationframework DotNet --applicationid "AAAAAAAA-AAAA-AAAA-AAAA-AAAAAAAAAAAA" --applicationmitigaterules "CWE89,CWE90" --applicationidentificationrules "CWE79,CWE89,CWE90" --applicationvectors "Database,WebRequest"

      The command below will delete the application with the application ID AAAAAAAA-AAAA-AAAA-AAAA-AAAAAAAAAAAA.
      SkyAnalyzer.Interface.Console.exe --credential k2FsaWNlQGNvbnRvc28uY29t --interface application --method delete --applicationid "AAAAAAAA-AAAA-AAAA-AAAA-AAAAAAAAAAAA"
  • Scan
    Enter the following in Command Prompt:
    SkyAnalyzer.Interface.Console.exe [--credential] --interface tools [--method][--applicationid][option [--applicationname][--applicationframework][-- applicationmitigaterules][--applicationidentificationrules][--applicationvectors]]
    • Methods:
      Method Description
      create Create a new scan.
      analyze Upload the source code archive and start the analysis.
      status Check if the scan has completed. True if the scan has completed (even if the scan has failed), False if the scan is ongoing.
      mitigate Download the secured source code of a completed scan.
      report Download the security report of a completed scan.
      delete Delete an existing scan.
    • Arguments:
      Option Description
      --credential Base64 encoded credentials. You can learn more about generating base64 encoded credentials in the Tools section.
      --scanid The GUID of the scan on AVM. AVM will use this GUID to identify the scan.
      --applicationid Required argument for method create.
      The GUID of the application on AVM. AVM will use this GUID to identify the application.
      --scanidentificationrules Optional argument for method create.
      The types of vulnerabilities to identify, separated by commas. For example, CWE79,CWE89,CWE90. If this argument is not present, settings from application will be used.
      This argument must be used with the argument scanmitigaterules. Also, rules in argument scanmitigaterules must also exist in this argument.
      --scanmitigaterules Optional argument for method create.
      The types of vulnerabilities to mitigate, separated by commas. For example, CWE89,CWE90. If this argument is not present, settings from application will be used.
      This argument must be used with the argument scanidentificationrules. Also, rules in this argument must also exist in argument scanidentificationrules.
      --scanvectors Optional argument for method create.
      The types of vectors considered potentially dangerous, separated by commas. For example, Database,WebRequest. If this argument is not present, settings from application will be used.
      --sourcecodearchive Required argument for method analyze.
      The path of the source code archive. For example, C:\Source.zip.
      --securedsourcecodearchive Required argument for method mitigate.
      The download path of the secured source code archive.
      --scanmitigationoption Required argument for method mitigate. Supported values are:
      1: including mitigation information
      2: including original source code
      3: including mitigation information and original source code
      If this argument is not present, no mitigation information and original source code are included.
      --report Required argument for method report.
      The download path of the security report.
      --reportformat Required argument for method report.
      The format of the report. Supported values are html and xml.
    • Example:
      The command below will create a scan with scan ID 55555555-5555-5555-5555-555555555555 under the application with application ID AAAAAAAA-AAAA-AAAA-AAAA-AAAAAAAAAAAA. Rules CWE79 and CWE89 are enabled for identification, with CWE89 enabled for mitigation. Database is the only vector considered potentially dangerous. k2FsaWNlQGNvbnRvc28uY29t is the base64 encoded string of the built-in account alice@contoso.com with the password password:
      SkyAnalyzer.Interface.Console.exe --credential k2FsaWNlQGNvbnRvc28uY29t --interface scan --method create --applicationid "AAAAAAAA-AAAA-AAAA-AAAA-AAAAAAAAAAAA" --scanid "55555555-5555-5555-5555-555555555555" --scanmitigaterules "CWE89" --scanidentificationrules "CWE79, CWE89" --scanvectors "Database"

      The command below will upload the source code archive at C:\Source.zip to the scan with scan ID 55555555-5555-5555-5555-555555555555 and start the analysis.
      SkyAnalyzer.Interface.Console.exe --credential k2FsaWNlQGNvbnRvc28uY29t --interface scan --method analyze --scanid "55555555-5555-5555-5555-555555555555" --sourcecodearchive "C:\Source.zip"

      To learn more about creating the source code archive of an application, click the following article number to view the article in the Lucent Sky Knowledge Base:
          KB342055 How to create a source code archive of an application for analysis and mitigation

      The command below will check the status of the scan with scan ID 55555555-5555-5555-5555-555555555555.
      SkyAnalyzer.Interface.Console.exe --credential k2FsaWNlQGNvbnRvc28uY29t --interface scan --method status --scanid "55555555-5555-5555-5555-555555555555"

      The command below will download the secured source code archive of the scan with scan ID 55555555-5555-5555-5555-555555555555 to C:\SecuredSource.zip. The secured source code contains mitigation information and original source code.
      SkyAnalyzer.Interface.Console.exe --credential k2FsaWNlQGNvbnRvc28uY29t --interface scan --method mitigate --scanid "55555555-5555-5555-5555-555555555555" --securedsourcecodearchive "C:\SecuredSource.zip" --scanmitigationoption 3

      The command below will download the security report in HTML format of the scan with scan ID 55555555-5555-5555-5555-555555555555 to C:\Report.zip.
      SkyAnalyzer.Interface.Console.exe --credential k2FsaWNlQGNvbnRvc28uY29t --interface scan --method report --scanid "55555555-5555-5555-5555-555555555555" --report "C:\Report.zip" --reportformat html

      The command below will delete the scan with scan ID 55555555-5555-5555-5555-555555555555.
      SkyAnalyzer.Interface.Console.exe --credential k2FsaWNlQGNvbnRvc28uY29t --interface scan --method delete --scanid "55555555-5555-5555-5555-555555555555"
