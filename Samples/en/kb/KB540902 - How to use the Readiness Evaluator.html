Introduction

This article outlines the steps of using Readiness Evaluator to ensure that your application is compatible with Lucent Sky AVM.

More Information

The Readiness Evaluator ensures an application is compatible with Lucent Sky AVM, by checking if the application is buildable and free of common issues that will cause it to be incompatible with Lucent Sky AVM.

Prerequisites

Before running Readiness Evaluator, you'll need to prepare your application's source code.

  • .NET Web Applications
    • In File Explorer, go to the root of the solution that contains the web application you want to secure. The root of the solution usually has the file ContosoWeb.sln, where ContosoWeb is the name of the solution.
    • Select all directories and files, except for those belonging to web applications other than the one you want to secure. You may also unselect non-code files, such as images and videos.
  • .NET Web Sites
    • In File Explorer, go to the root of the web site you wish to secure. The root of the web site usually has the file web.config.
    • Select all files under the root. You may unselect non-code files, such as images and videos.
  • Java Web Applications
    • In File Explorer, go to the root of the project that contains the web application you wish to secure. The root of the solution usually has the source directory (either java or src) and the web directory (usually webapp or WebContent).
    • Select all directories and files. You may also unselect non-code files, such as images and videos.
    • For more details on ensuring that your application directory structure is compatible, see: KB572585 - How to prepare Java source code for analysis by Lucent Sky AVM.
  • Java Web Sites
    • In File Explorer, go to the root of the web site you wish to secure. The root of the web site usually has the directory WEB-INF.
    • Select all files under the root. You may unselect non-code files, such as images and videos.
    • For more details on ensuring that your application directory structure is compatible, see: KB572585 - How to prepare Java source code for analysis by Lucent Sky AVM.

Readiness Evaluator requires .NET Framework 4.0 or newer to run. You can download the latest version of .NET Framework compatible with your system from Windows Update.

Using Readiness Evaluator

  1. Download Readiness Evaluator and unzip to a directory.
  2. Double click the icon named Run.bat.
  3. Enter your framework: DotNet, Java or PHP.
  4. Enter your application source root prepared in step 1.
  5. Enter the scan arguments if needed.
  6. Wait for the Readiness Evaluator as it attempts to build the project, it'll return a message indicating if the build was successful.

Troubleshooting

Readiness Evaluator will produce a Logs directory to assist the troubleshooting of build errors.

  • For a .NET application:
    • The tool will create an MSBuild.log in the Logs directory, and there're the messages during the building stage. If you get any error message in building stage, you can get more information from this file.
    • The tool will create a Precompilation.log in the Logs directory, and there're the messages during the precompilation stage. If you get any error message in precompile stage, you can get more information from this file.
  • For a Java application:
    • If you receive a message that the project could not be compiled, then navigate to the Logs directory, and open Ant.log. Look for an occurrence of any text string that reads error. Error messages corresponding to [javac] denote Java files that are failing to compile; if a JSP file fails to compile, you will see an error message either underneath jspc or corresponding to a filename ending in _jsp.java corresponding to a line starting with [javac].
    • If the log files indicate that a .class file could not be found, the application (or projects it depends on) may be referencing libraries that are present in your system and are part of the project's classpath during compilation, but which are not present in the directory of the web application that the Readiness Evaluator is scanning. Identify these libraries, and copy them to the WEB-INF\lib folder of the web application. Then run Readiness Evaluator again.
    • Lucent Sky AVM supports application with a compliance level of 1.5 or higher, but defaults to compliance level 1.6. Readiness Evaluator assumes that source code is compliant with Java 1.6. If the Java compiler is failing to properly compile the project because the compliance level does not correspond to that of your project, contact Lucent Sky support for changing the default compliance level.
    • If the project builds without error within your local IDE and the log files within the Readiness Evaluator still contain errors indicating that some files are not building, and if the faulting files are encoded with your native encoding (such as Big-5 for example), try converting the faulting source files to UTF-8 (without a BOM) and run Readiness Evaluator again.
