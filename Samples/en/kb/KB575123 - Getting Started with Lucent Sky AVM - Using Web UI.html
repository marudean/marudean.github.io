Introduction

Lucent Sky AVM is a system that helps developers and other stakeholders identify and automatically mitigate vulnerabilities in an application's source code. Lucent Sky AVM is composed of three components, CLEAR Engine (the core analysis and mitigation engine), Web UI, and Developer UIs. The Developer UIs include CLI, Visual Studio Extension and Eclipse plug-in.

More Information

Prerequisites for Developers

Developers should be familiar with the following tools:

  • C#, Java or VB.NET languages
    • This guide uses C# (which has a similar syntax with Java and VB.NET) in its examples and screenshots.

Using Web UI

In this section, you will see:

  • A sample procedure for analyzing and mitigating an application with the Web UI

At the end of this section, you will be able to:

  • Create a zip file containing the source code of an application
  • Use Web UI to scan an application for vulnerabilities and automatically fixing them
  • Review the identified vulnerabilities and their Instant Fix on the Web UI
  • Download the secured source code of an application from the Web UI
  • Download the security report of an application from the Web UI

To create a zip file of an application

  • .NET Web Applications
    1. In File Explorer, go to the root of the solution that contains the web application you want to secure. The root of the solution usually has the file ContosoWeb.sln, where ContosoWeb is the name of the solution.
    2. Select all directories and files, except for those belong to web applications other than the one you want to secure. You may also unselect non-code files, such as images and videos.
    3. Right click on the selected files, move the mouse cursor over Send to, then click Compressed (zipped) folder. Or use your preferred archive program (such as 7-Zip) to create an archive in zip format.
  • .NET Web Sites
    1. In File Explorer, go to the root of the web site you wish to secure. The root of the web site usually has the file web.config.
    2. Select all files in under the root. You may unselect non-code files, such as images and videos.
    3. Right click on the selected files, move the mouse cursor over Send to, then click Compressed (zipped) folder. Or use your preferred archive program (such as 7-Zip) to create an archive in zip format.
  • Java Web Applications
    1. In File Explorer, go to the root of the project that contains the web application you wish to secure. The root of the solution usually has the source directory (either java or src) and the web directory (usually webapp or WebContent).
    2. Select all directories and files. You may also unselect non-code files, such as images and videos.
    3. Right click on the selected files, move the mouse cursor over Send to, then click Compressed (zipped) folder. Or use your preferred archive program (such as 7-Zip) to create an archive in zip format.
  • Java Web Sites
    1. In File Explorer, go to the root of the web site you wish to secure. The root of the web site usually has the directory WEB-INF.
    2. Select all files in under the root. You may unselect non-code files, such as images and videos.
    3. Right click on the selected files, move the mouse cursor over Send to, then click Compressed (zipped) folder. Or use your preferred archive program (such as 7-Zip) to create an archive in zip format.

To scan an application for vulnerabilities

  1. Go to the Web UI in your browser, and then sign in with your credentials.
  2. Under the Action Bar on the left, click New Application. Enter a descriptive name of the application to be scanned in Application Name, select the application framework, and then click Create. You can also optionally change the default analysis and mitigation parameters for this application by clicking More Options. You can change these parameters at a later stage as well.
  3. Select Static Code Analysis for Analysis Method. Select Standard Analysis for Analysis Mode for a standard analysis or Comprehensive Analysis for a deep-level, thorough analysis. Click Browse… and select the zip file containing the source code of the application. You can also optionally change the default analysis and mitigation parameters for this application by clicking More Options. Finally, click Upload.
  4. The web page will refresh itself. An Application Box will be shown with the progress of the scan. There are 6 stages:
    • Building
    • Compiling
    • Analyzing (S-1)
    • Analyzing (S-2)
    • Analyzing (S-3)
    • Analyzing (S-4)
  5. Depends on the size and complexity of the application, the scan can take anytime from a few seconds to a few hours to complete. Under most circumstances, a Lucent Sky AVM server is capable of finishing scanning a 100,000 lines application in under 10 minutes.
  6. Once the scan is completed, the Application Box will show the number of total vulnerabilities, the number of vulnerabilities that have been automatically mitigated, as well as the number of those couldn't be automatically mitigated.

To review the identified vulnerabilities and their Instant Fix

  1. Go to the Web UI in your browser, and then sign in with your credentials.
  2. Click the Application Box of the application you want to review. You will be taken to the Scan Summary page.
  3. The Scan Summary page has three parts: Efficiency Snapshot, Vulnerabilities and Additional Information.
    • The Efficiency Snapshot shows the mitigation cost and programming hours saved by automatically mitigating the vulnerabilities in this application.
    • The Vulnerabilities pie chart shows the distribution of vulnerability categories, as well as the number of vulnerabilities in each category. Clicking a vulnerability slice or a vulnerability box next to the pie chart will take you to the Vulnerability List page.
    • Additional Information shows information about this scan, as well as the number of vulnerabilities compared with the previous scan of the same application.
  4. The Vulnerability List page is a list of all the vulnerabilities in that category. Clicking a vulnerability will take you to the Vulnerability Details page.
  5. The title of the Vulnerability Details page shows the category, the file and line number of the vulnerability. The Statements shows the flow of the vulnerability in the application, and Instant Fix shows how the vulnerability can be mitigated by replacing the source code at the file and line indicated with the Instant Fix.

To download the secured software of an application of an application

  1. Go to the Web UI in your browser, and then sign in with your credentials.
  2. Click the Application Box of the application you want to review. You will be taken to the Scan Summary page.
  3. Under the Action Bar on the left, click Mitigate. In the pop-up dialog, choose if you want to include mitigation information and original source code in the generated source code as comments. The generated source code might be similar to
    {% highlight csharp %} Response.Write(HtmlEncode(Request["foo"])); // CWE79 (High); Original source code: Response.Write(Request["Foo"]); {% endhighlight %}
    where CWE79 (High) is the mitigation information and Original source code: Response.Write(Request["Foo"]); is the original source code.
  4. Click Mitigate and download to download the secured source code.

To download the security report of an application of an application

  1. Go to the Web UI in your browser, and then sign in with your credentials.
  2. Click the Application Box of the application you want to review. You will be taken to the Scan Summary page.
  3. Under the Action Bar on the left, click Report. In the pop-up dialog, choose the format of the report, then click Generate and download.
