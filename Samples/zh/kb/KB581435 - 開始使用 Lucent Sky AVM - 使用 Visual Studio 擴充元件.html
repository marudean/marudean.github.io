介紹

Lucent Sky AVM 是一個幫助開發者及相關人員偵測程式碼弱點並自動修正的系統。Lucent Sky AVM 包括了 CLEAR Engine（分析及修正引擎）、 Web UI 及 Developer UIs 。其中 Developer UIs 包括：Lucent Sky AVM CLI、Lucent Sky AVM Visual Studio 擴充元件和 Eclipse 外掛元件。

開發者需求

開發者應熟悉下列的工具：

  • C#, Java 或 VB.NET languages
    • 這篇指南的範例及截圖都是使用C#（C# 的語法跟 Java 和 VB.NET 都非常相似）。
  • Visual Studio 2012 或 Visual Studio 2013

建立 Visual Studio 和 AVM 主機間通訊的前置作業

  1. 關閉所有正在執行的 Visual Studio 視窗後，以文字編輯器開啟 Visual Studio 的組態設定檔（檔案通常位於：C:\Program Files (x86)\Microsoft Visual Studio 11.0\Common7\IDE\devenv.exe.config，其中 11.0 是 Visual Studio 的版本）。修改 system.serviceModel 部分以允許 Visual Studio 和 AVM 主機進行通訊

若你希望了解更多關於修改 Visual Studio 的組態設定檔的細節，請參考 Lucent Sky 知識庫：
    KB390837 Administration of Lucent Sky AVM Visual Studio Extension

在開發環境上安裝 AVM Visual Studio 擴充元件

  1. 在 AVM Visual Studio 擴充元件安裝檔案（CLEAR_VSIX.vsix）上點擊滑鼠左鍵兩次。
  2. 依照安裝精靈的指示完成安裝。

若你希望得知更多關於AVM Visual Studio 擴充元件安裝及組態設定等相關資訊，請參考 Lucent Sky 知識庫：

使用 AVM Visual Studio 擴充元件進行應用程式中的弱點的掃描及修正

  1. 若是網頁應用程式，以 Visual Studio 開啟專案檔；若是網站，以 Visual Studio 開啟網站。
  2. 在 [方案總管] 中，以右鍵點選想要分析的專案或網站，接著選擇 [Analyze Security Issues] 項目。
  3. 接著會跳出 [Analyze Security Issues] 視窗，點選 [Scan] 以開始分析。你也可以針對不同弱點類型設定個別的處理方式及偵測來源類型進行相關設定。
  4. 此時會出現 [Analysis Results] 視窗，當掃描正在進行的時候，這個視窗將會顯示階段狀態並且自動更新。你也可以點選右上角的 [Refresh] 按鈕以取得掃描的最新狀態，掃描共有六個階段：
    • 建構中
    • 編譯中
    • 分析中 (S-1)
    • 分析中 (S-2)
    • 分析中 (S-3)
    • 分析中 (S-4)
  5. 一旦分析完成後，弱點將會顯示在 [Analysis Results] 視窗，展開弱點分類將會顯示該分類下的所有弱點。再向下展開單一弱點則會顯示該弱點的所有陳述和立即修正的語法（若無法進行自動修正，就不會顯示立即修正的資訊）。雙鍵點選 陳述 或 立即修正 ，將會自動跳至該弱點的行數。
  6. 以右鍵點擊立即修正並選擇 [Mitigate] ，該語法將會直接自動被 Visual Studio 套用且接受。
