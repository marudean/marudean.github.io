介紹

Lucent Sky AVM 是一個幫助開發者及相關人員偵測程式碼弱點並自動修正的系統。Lucent Sky AVM 包括了 CLEAR Engine（分析及修正引擎）、 Web UI 及 Developer UIs 。其中 Developer UIs 包括：Lucent Sky AVM CLI、Lucent Sky AVM Visual Studio 擴充元件和 Eclipse 外掛元件。

開發者需求

開發者應熟悉下列的工具：

  • C#、Java 或 VB.NET
    • 這篇指南的範例及截圖都是使用 C#（C# 的語法跟 Java 和 VB.NET 都非常相似）。

使用 Web UI

本段你將會看到：

  • 如何透過 Web UI 來偵測及修正應用程式的步驟

本段結束時，你會知道如何：

  • 建立一個包含了你的應用程式程式碼的壓縮檔案
  • 使用 Web UI 來掃描你的應用程式弱點並自動修正
  • 查看 Web UI 掃描的弱點及修正結果
  • 透過 Web UI 下載你應用程式修正過後的程式碼
  • 透過 Web UI 下載你應用程式的資安結果報告

建立一個應用程式的壓縮檔案

  • .NET網頁應用程式（.NET Web Applications）
    1. 以檔案總管開啟你想掃描的網頁應用程式方案的根目錄，也就是包含 ContosoWeb.sln 檔案的目錄，其中 ContosoWeb 是你網頁應用程式方案的名稱。
    2. 選取全部的網頁應用程式資料夾及檔案，但你可以排除不希望掃描的檔案、或是不包含程式碼的檔案，例如圖片或影片檔。
    3. 以滑鼠右鍵點選選擇的檔案，將滑鼠移到 [傳送到(N)] 標籤，並選擇 [壓縮的(Zipped) 資料夾]。或者你也可以使用你喜歡的壓縮軟體，例如 7-Zip 或是 WinRAR 將檔案壓縮成 zip 格式。
  • .NET 網站
    1. 以檔案總管開啟你想掃描的網站的根目錄，這個目錄通常包含 web.config 檔案。
    2. 選取全部根目錄中包含的資料夾及檔案，但你可以排除不包含程式碼的檔案，例如圖片或影片檔。
    3. 以滑鼠右鍵點選選擇的檔案，將滑鼠移到 [傳送到(N)] 標籤，並選擇 [壓縮的(Zipped) 資料夾]。或者你也可以使用你喜歡的壓縮軟體，例如 7-Zip 或是 WinRAR 將檔案壓縮成 zip 格式。
  • Java 網頁應用程式（Java Web Applications）
    1. 以檔案總管開啟你想掃描的網頁應用程式的根目錄，這個目錄通常包含來源資料夾（通常命名為 java 或 src）及網頁資料夾（通常命名為 webapp 或 WebContent）。
    2. 選取全部根目錄中包含的資料夾及檔案，但你可以排除不包含程式碼的檔案，例如圖片或影片檔。
    3. 以滑鼠右鍵點選選擇的檔案，將滑鼠移到 [傳送到(N)] 標籤，並選擇 [壓縮的(Zipped) 資料夾]。或者你也可以使用你喜歡的壓縮軟體，例如 7-Zip 或是 WinRAR 將檔案壓縮成 zip 格式。
  • Java 網站
    1. 以檔案總管開啟你想掃描的網站的根目錄，這個目錄通常包含資料夾 WEB-INF。
    2. 選取全部根目錄中包含的資料夾及檔案，但你可以排除不包含程式碼的檔案，例如圖片或影片檔。
    3. 以滑鼠右鍵點選選擇的檔案，將滑鼠移到 [傳送到(N)] 標籤，並選擇 [壓縮的(Zipped) 資料夾]。或者你也可以使用你喜歡的壓縮軟體，例如 7-Zip 或是 WinRAR 將檔案壓縮成 zip 格式。

掃描應用程式弱點

  1. 以瀏覽器開啟 Web UI 並以你的帳號登入。
  2. 點選左側 [新應用程式] 後，請輸入應用程式名稱並選擇框架類型後，點選建立按鈕。或者視你的需要，也可以先點選 [更多選項] ，針對系統分析及修正的預設值進行設定，或者你也可以選擇稍後再進行這些設定。
  3. 接著在 [分析方式] 請選擇 [靜態程式碼分析]，至於 [分析模式] 部分，你可以選擇 [標準分析] 進行一個標準模式的分析、也可以選擇 [完整分析] 進行一個較完整且深入的分析。接著請在 [程式碼壓縮檔] 部分選擇上傳你程式程式碼的 zip 檔案。在此處你也可以點選 [更多選項] ，針對系統分析及修正的設定值進行修改。完成相關設定後請點選 [上傳] 按鈕。
  4. 網頁將會自動刷新，此時會出現應用程式方框顯示掃描的進度及情況，掃描共分為六個階段：
    • 建構中
    • 編譯中
    • 分析中 (S-1)
    • 分析中 (S-2)
    • 分析中 (S-3)
    • 分析中 (S-4)
  5. 掃描所需的時間大約在幾秒至幾小時都有可能，需視應用程式的大小及複雜度而定。在大多數的情況下，10萬行左右的程式碼會在10分鐘內完成掃描。
  6. 當掃描結束時，應用程式方框將會顯示總弱點數、自動修正弱點數及未自動修正的弱點數目。

查看 Web UI 偵測到的弱點及立即修正的結果

  1. 以瀏覽器開啟 Web UI 並以你的帳號登入。
  2. 點選你想檢視的應用程式方框，將會進入 [掃描概要] 頁面。
  3. [掃描概要] 頁面分為三個部分：效益概觀、弱點資訊及附加資訊。
    • 效益概觀 顯示這個應用程式節省的修正成本及節省的開發時間。
    • 弱點資訊 圓餅圖顯示了弱點種類的分佈情況、並顯示各種類的弱點數量。若點選圓餅圖上該弱點種類、或右側該分類區塊，則會進入該種類的 [弱點清單] 頁面。
    • 附加資訊 顯示此次掃描的相關資訊並比較這個應用程式前後次掃描的弱點數量變化。
  4. [弱點清單] 頁面會列出該種類所有的弱點，點選單一弱點則可以進入 [弱點明細] 頁面。
  5. [弱點明細] 頁面頁面會在標題顯示該弱點的分類、檔案名稱及行數、 [陳述] 部分會顯示弱點在程式碼中的流程，至於 [立即修正] 部分則顯示建議修正的檔案位置、行號及建議用來取代該行的程式碼。

下載應用程式修正過後的程式碼

  1. 以瀏覽器開啟 Web UI 並以你的帳號登入。
  2. 點選你想檢視的應用程式方框，將會進入 [掃描概要] 頁面。
  3. 點選畫面左側的 [修正] 按鈕，將會跳出一個視窗，你可以選擇你是否需要以註解方式加入修正資訊及原始程式碼資訊。若你選擇加入相關資訊，會看到類似這樣的結果：
    {% highlight csharp %} Response.Write(HtmlEncode(Request["foo"])); // CWE79 (High); Original source code: Response.Write(Request["Foo"]); {% endhighlight %}
    其中 CWE79 (High) 是修正資訊、而 Original source code: Response.Write(Request["Foo"]); 則是原始程式碼。
  4. 點選 [修正並下載] 按鈕，將會下載應用程式修正過後的程式碼。

下載應用程式的資訊安全報告

  1. 以瀏覽器開啟 Web UI 並以你的帳號登入。
  2. 點選你想檢視的應用程式方框，將會進入 [掃描概要] 頁面。
  3. 點選畫面左側的 [報告] 按鈕，將會跳出一個視窗，選擇要下載的檔案格式後，點選 [產生並下載] 按鈕來下載報告。
