介紹

Lucent Sky AVM 是一個幫助開發者及相關人員偵測程式碼弱點並自動修正的系統。Lucent Sky AVM 包括了 CLEAR Engine（分析及修正引擎）、 Web UI 及 Developer UIs 。其中 Developer UIs 包括：Lucent Sky AVM CLI、Lucent Sky AVM Visual Studio 擴充元件和 Eclipse 外掛元件。

更多資訊

開發者需求

開發者應熟悉下列的工具：

  • C#、Java 或 VB.NET
    • 這篇指南的範例及截圖都是使用C#（C# 的語法跟 Java 和 VB.NET 都非常相似）。
  • Bash, Command Prompt 或 Windows PowerShell

使用 CLI

本段你將會看到：

  • 示範如何安裝及配置 AVM CLI
  • 示範如何利用 AVM CLI 來偵測及修正應用程式

本段結束時，你會知道如何：

  • 在開發環境安裝及配置 AVM CLI
  • 透過 AVM CLI 來掃描應用程式以偵測弱點並立即修正
  • 透過 AVM Web CLI 來下載你應用程式修正過後的程式碼
  • 透過 AVM Web CLI 來下載你應用程式的資訊安全報告

安裝及配置AVM CLI

  1. 在 AVM CLI 安裝檔案（CLEAR_CLI.exe）上點擊滑鼠左鍵兩次，檔案將會自動解壓縮到目錄 C:\Program Files\Lucent Sky\CLEAR CLI 並新增一個 CLEAR_CLI 資料夾（你也可以自行指定目標資料夾）。
  2. 開啟 C:\Program Files\Lucent Sky\CLEAR CLI 或你在上一步自行指定的資料夾位置，並以文字編輯器開啟 SkyAnalyzer.Interface.Console.exe 檔案。
  3. 以你 AVM 主機的FQDN、IP位置或電腦名稱取代 clear.contoso.com 這段文字，並將檔案儲存後關閉文字編輯器。

AVM CLI 和 AVM主機預設使用不安全的連線，若你希望能改為採用安全的連線，請參考 Lucent Sky 知識庫：
    KB448674 Administration of Lucent Sky AVM CLI

透過 AVM CLI 來掃描應用程式以偵測弱點並立即修正

AVM CLI共分為三大類功能：「tools」、「application」及「scan」，這將是呼叫CLI時選項「interface」的參數內容，接下來將逐項說明。

  • Tools
    在命令提示字元中輸入下列文字：
    SkyAnalyzer.Interface.Console.exe --interface tools [--method][--value]
    • 方法：
      方法 說明
      base64encode 取得 base64 編碼後的憑證。配合 value 引數使用。
    • 引數：
      選項 說明
      --value base64encode 方法所需要的引數。
      格式為：「登入AVM的電子郵件 密碼 帳號類型」。若是 AVM 內建的帳號，在帳號類型輸入 Sql 」；若是組織帳號，請輸入 ActiveDirectory 。
    • 範例：
      下列命令會生成內建帳號 alice@contoso.com 、密碼為 password 的base64編碼憑證：
      SkyAnalyzer.Interface.Console.exe --interface tools --method base64encode --value "alice@contoso.com password Sql"
  • Application
    在命令提示字元中輸入下列文字：
    SkyAnalyzer.Interface.Console.exe [--credential] --interface tools [--method][--applicationid][option [--applicationname][--applicationframework][-- applicationmitigaterules][--applicationidentificationrules][--applicationvectors]]
    • 方法：
      方法 說明
      create 建立一個新的應用程式。
      delete 刪除一個已存在的應用程式。
    • 引數：
      選項 說明
      --credential 以Base64編碼的憑證資訊。你可以在「Tools」段落找到生成憑證的相關資訊。
      --applicationid AVM 中應用程式的 GUID。AVM 將會使用這個 GUID 來辨識應用程式。
      --applicationname create 方法所需要的引數。
      應用程式的名稱。
      --applicationframework create 方法所需要的引數。
      應用程式的框架類型，目前支援 DotNet、Java 及 PHP.
      --applicationidentificationrules create 方法的選擇性引數。
      應用程式要偵測的弱點類別，以逗號串聯，例如：CWE79,CWE89,CWE90。若此引數不存在則會偵測現存所有的弱點類別。
      這個引數必需和 applicationmitigaterules 引數一起使用。同時，applicationmitigaterules 引數中的弱點類別一定要存在於這個引數中。
      --applicationmitigaterules create 方法的選擇性引數。
      應用程式要修正的弱點類別，以逗號串聯，例如：CWE89,CWE90。若此引數不存在則會修正現存所有的弱點類別。
      這個引數必需和 applicationidentificationrules 引數一起使用。同時，這個引數中的弱點類別一定要存在於 applicationidentificationrules 引數中。
      --applicationvectors create 方法的選擇性引數。
      不信任的資料來源，以逗號串聯，例如：Database,WebRequest。若此引數不存在則會偵測現存所有的資料來源。
    • 範例：
      下列命令會新增一個DotNet（.NET）應用程式、名稱為 ContosoWeb、應用程式的代號為 AAAAAAAA-AAAA-AAAA-AAAA-AAAAAAAAAAAA、偵測 CWE79, CWE89 及 CWE90 規則、修正 CWE89 and CWE90 規則，並只將 Database 及 WebRequest 視為不被信任的資料來源。其中 k2FsaWNlQGNvbnRvc28uY29t 為內建帳號 alice@contoso.com、密碼為 password 的 base64 編碼憑證。
      SkyAnalyzer.Interface.Console.exe --credential k2FsaWNlQGNvbnRvc28uY29t --interface application --method create --applicationname "ContosoWeb" --applicationframework DotNet --applicationid "AAAAAAAA-AAAA-AAAA-AAAA-AAAAAAAAAAAA" --applicationmitigaterules "CWE89,CWE90" --applicationidentificationrules "CWE79,CWE89,CWE90" --applicationvectors "Database,WebRequest"

      下列命令會刪除代號 AAAAAAAA-AAAA-AAAA-AAAA-AAAAAAAAAAAA 的應用程式。
      SkyAnalyzer.Interface.Console.exe --credential k2FsaWNlQGNvbnRvc28uY29t --interface application --method delete --applicationid "AAAAAAAA-AAAA-AAAA-AAAA-AAAAAAAAAAAA"
  • Scan
    在命令提示字元中輸入下列文字：
    SkyAnalyzer.Interface.Console.exe [--credential] --interface tools [--method][--applicationid][option [--applicationname][--applicationframework][-- applicationmitigaterules][--applicationidentificationrules][--applicationvectors]]
    • 方法：
      方法 說明
      create 新增一個新的掃描。
      analyze 上傳程式碼檔案並開始進行分析。
      status 查詢掃描是否已完成。若已完成將會回傳 True （即使掃描失敗了）、若否則會回傳 False 。
      mitigate 下載已完成掃描及弱點修正的程式碼。
      report 下載已完成掃描的資訊安全報告。
      delete 刪除一個已存在的掃描。
    • 引數:
      選項 說明
      --credential 以 Base64 編碼的憑證資訊，你可以在「Tools」段落找到生成憑證的相關資訊。
      --scanid AVM 中掃描的 GUID，AVM 將會使用這個 GUID 來辨識掃描。
      --applicationid create 方法所需要的引數。
      AVM 中應用程式的 GUID，AVM 將會使用這個 GUID 來辨識應用程式。
      --scanidentificationrules create 方法的選擇性引數。
      掃描要偵測的規則，以逗號串聯，例如：CWE79,CWE89,CWE90。若此引數不存在則會繼承應用程式的設定。
      這個引數必需和 scanmitigaterules引數一起使用。同時，scanmitigaterules 引數中的弱點類別一定要存在於這個引數中。
      --scanmitigaterules create 方法的選擇性引數。
      掃描要修正的規則，以逗號串聯，例如：CWE89,CWE90。若此引數不存在則會繼承應用程式的設定。
      這個引數必需和 scanidentificationrules引數一起使用。同時，這個引數中的弱點類別一定要存在於 scanidentificationrules 引數中。
      --scanvectors create 方法的選擇性引數。
      掃描不信任的資料來源，以逗號串聯，例如：Database,WebRequest。若此引數不存在則會繼承應用程式的設定。
      --sourcecodearchive analyze 方法所需要的引數。
      程式碼壓縮檔的路徑，例如：C:\Source.zip。
      --securedsourcecodearchive mitigate 方法所需要的引數。
      下載修正後程式碼壓縮檔的路徑。
      --scanmitigationoption mitigate 方法的選擇性引數。 Supported values are:
      1: 包含修正資訊
      2: 包含原始程式碼
      3: 包含修正資訊及原始程式碼
      若此引數不存在，則不包含修正資訊及原始程式碼。
      --report report 方法所需要的引數。
      下載資訊安全報告的路徑。
      --reportformat report 方法所需要的引數。
      資訊安全報告的格式，目前支援 html 及 xml。
    • 範例:
      下列命令會在應用程式 AAAAAAAA-AAAA-AAAA-AAAA-AAAAAAAAAAAA 新增一個代號為 55555555-5555-5555-5555-555555555555 的掃描，偵測 CWE79 及 CWE89 規則、修正 CWE89 規則，並只將 Database 視為不被信任的資料來源。其中 k2FsaWNlQGNvbnRvc28uY29t 為內建帳號 alice@contoso.com、密碼為 password 的憑證。
      SkyAnalyzer.Interface.Console.exe --credential k2FsaWNlQGNvbnRvc28uY29t --interface scan --method create --applicationid "AAAAAAAA-AAAA-AAAA-AAAA-AAAAAAAAAAAA" --scanid "55555555-5555-5555-5555-555555555555" --scanmitigaterules "CWE89" --scanidentificationrules "CWE79, CWE89" --scanvectors "Database"

      下列命令會將位於 C:\Source.zip 的程式碼壓縮檔上傳到掃瞄 55555555-5555-5555-5555-555555555555 並開始進行分析。
      SkyAnalyzer.Interface.Console.exe --credential k2FsaWNlQGNvbnRvc28uY29t --interface scan --method analyze --scanid "55555555-5555-5555-5555-555555555555" --sourcecodearchive "C:\Source.zip"

      若你想獲取更多建立應用程式程式碼壓縮檔的相關資訊，請參考 Lucent Sky 知識庫：
          KB342055 How to create a source code archive of an application for analysis and mitigation

      下列命令會確認代號 55555555-5555-5555-5555-555555555555 的掃瞄目前進行的狀態。
      SkyAnalyzer.Interface.Console.exe --credential k2FsaWNlQGNvbnRvc28uY29t --interface scan --method status --scanid "55555555-5555-5555-5555-555555555555"

      下列命令會將代號 55555555-5555-5555-5555-555555555555 的掃描的修正後的檔案程式碼下載至 C:\SecuredSource.zip 路徑。其中修正後的程式碼檔案內包含修正資訊及原始程式碼的資訊。
      SkyAnalyzer.Interface.Console.exe --credential k2FsaWNlQGNvbnRvc28uY29t --interface scan --method mitigate --scanid "55555555-5555-5555-5555-555555555555" --securedsourcecodearchive "C:\SecuredSource.zip" --scanmitigationoption 3

      下列命令會將代號 55555555-5555-5555-5555-555555555555 的掃描的 HTML 格式掃瞄報告下載至 C:\Report.zip 路徑。
      SkyAnalyzer.Interface.Console.exe --credential k2FsaWNlQGNvbnRvc28uY29t --interface scan --method report --scanid "55555555-5555-5555-5555-555555555555" --report "C:\Report.zip" --reportformat html

      下列命令會刪除代號 55555555-5555-5555-5555-555555555555 的掃瞄。
      SkyAnalyzer.Interface.Console.exe --credential k2FsaWNlQGNvbnRvc28uY29t --interface scan --method delete --scanid "55555555-5555-5555-5555-555555555555"
